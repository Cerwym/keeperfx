# Project Infrastructure - PSVita Hardware Acceleration

## Build System

**Type:** CMake + vcpkg dependency management

**Primary Target:** PSVita (arm-vita-eabi cross-compiler)  
**Secondary Targets:** Windows x86 (MinGW32), Linux x64

## Critical Files

| File | Purpose |
|------|---------|
| `CMakePresets.json` | CMake build presets for different toolchains |
| `CMakeLists.txt` | Main CMake build definition |
| `vcpkg.json` | Dependency manifest for vcpkg |
| `.vscode/tasks.json` | VS Code build/deploy tasks |

## Build Commands

```bash
# Configure
cmake --preset x86-MinGW32-Debug
cmake --preset <vita-preset>

# Build
cmake --build --preset x86-MinGW32-Debug
cmake --build --preset <vita-preset>
```

## Vita Build Tasks (from .vscode/tasks.json)

- `Build Vita Debug` - Debug build
- `Build Vita RelDebug` - Release with debug symbols
- `Build Vita Release` - Optimized release build
- `Deploy Vita eboot.bin` - FTP upload executable
- `Deploy & Launch Vita` - Upload and restart app
- `Reboot Vita` - Reboot device
- `Fetch Vita Logs` - Download crash dumps

## Toolchains

| Target | Compiler | Preset |
|--------|----------|--------|
| Windows x86 | arm-none-eabi-gcc (MinGW) | x86-MinGW32-Debug/Release |
| Linux x64 | gcc/clang | Linux preset |
| PSVita | arm-vita-eabi-gcc | Vita-Debug/RelDebug/Release |

## CRITICAL RULE

**ALL CODE CHANGES MUST COMPILE against arm-vita-eabi-g++**

Never commit without verifying Vita compilation. This is the primary target platform.
